# Admin User Tools

These tools create parent/trainer accounts from your local machine.

Recommended for adding parents and children:

```bash
npm run user:ui:staging
npm run user:ui:production
```

Fallback terminal wizard:

```bash
npm run user:create:staging
npm run user:create:production
```

Physiotherapist report wizard:

```bash
npm run report:create:staging
npm run report:create:production
```

## Local UI

The UI command prints a URL like:

```text
http://127.0.0.1:4317/?token=...
```

Open it in your browser.

What it does:

- runs only on `127.0.0.1`
- requires the random token in the URL
- gets `DATABASE_URL` from `vercel env run`
- never creates a local env file
- generates a temporary password
- forces password change on first login
- creates the user, children, and results in one transaction

The UI has one simple table. One row means one child/result row. To add multiple
results for the same child, repeat the same child name, birth year, and group on
multiple rows.

Leave result fields empty to create a child without a result.

## Vercel Setup

If the repo is not linked to Vercel yet:

```bash
npm run vercel:link
```

The user tools then run through:

```bash
vercel env run -e preview -- ...
vercel env run -e production -- ...
```

That injects Vercel env vars into the command process without writing them to
`.env.local`.

## Database Migrations

Apply committed migrations before using scripts that depend on new tables:

```bash
npm run db:migrate:staging
npm run db:migrate:production
```

Use staging first. Production migrations should be run intentionally, after
checking the printed command and deployment state.

## Production Rules

Production is intentionally strict:

- check the printed database host before using the tool
- type the exact confirmation text before saving
- temporary passwords are generated by the tool
- users must change password on first login
- send email and temporary password through separate channels

## Physiotherapist Reports

The report wizard adds one report to one child.

It asks for:

- child
- report date
- observations, one per line
- optional note about what changed since the previous report
- recommendations, one per line

Use the exact wording from the physiotherapist where possible.

## Do Not Commit

Do not commit:

- `.env*`
- `.vercel/`
- temporary password lists
